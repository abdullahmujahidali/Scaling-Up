# Starting point: Django backend engineer, weak across all system design areas

Learner is a strong Python/Django backend engineer who self-rates as weak/uncertain across all four system design areas (databases, scaling/caching, distributed theory, async/messaging). Mission blends three goals — pass design interviews, architect better at work, get promoted — so lessons should serve all three at once. See [[MISSION.md]].

**Implications for teaching:**
- Use the Django background as a bridge to lower difficulty during knowledge acquisition (they already have intuition for databases, request/response, ORM, migrations).
- Start at fundamentals but move fast where Django gives a head start.
- Lesson 1 taught the 5-step interview framework (Scope → Estimate → Draw → Deep-dive → Trade off) as the scaffold for the whole course. Not yet demonstrated understanding — coverage only. Wait for evidence (a dry-run attempt or correct application) before writing a record promoting it.
